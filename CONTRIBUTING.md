# Team Norms

## Communication

- We will communicate primarily through Discord.
- We will respond to team messages within 24 hours.
- Standup updates will be posted regularly.

## Meetings

- We will attend all scheduled standup meetings.
- If someone cannot attend, they must notify the team in advance.

## Task Ownership

- Each task must have an assigned owner.
- If a task cannot be completed on time, the owner must inform the team early.

## Conflict Resolution

Conflict is expected and will be handled professionally.

If a teammate misses two consecutive meetings or does not meaningfully participate:

1. The team will first reach out privately to understand the issue.
2. We will attempt to resolve the problem internally.
3. If unresolved before the next standup:
   - The issue will be reported to the project admins.
   - Other team members will take over the responsibilities.
   - Updates will continue to be shared publicly.
4. If the teammate later re-engages, the team will support reintegration.
5. The team will not delay progress for more than two standups.

## Code Contribution

- All code must be submitted via pull request.
- At least one teammate must review before merging.
